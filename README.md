## Архітектурні рішення

### Використані технології
- **Vue 3 (Composition API + `<script setup>`)** — для побудови компонентів із більш читабельною та перевикористовуваною логікою.
- **TypeScript** — для статичної типізації, покращення підтримуваності коду та зменшення кількості помилок під час розробки.
- **Pinia** — як централізоване сховище стану для роботи з проектами та задачами.
- **PrimeVue** — для швидкого створення UI з готових компонентів (таблиці, модальні вікна, форми, кнопки тощо).
- **Axios** — для взаємодії з MockAPI.
- **Vuelidate** — для валідації форм створення та редагування проектів і задач.

## Структура проєкту та поділ на компоненти

### `components/`
Компоненти розділені за функціональним призначенням:

- `base/` — базові перевикористовувані компоненти (наприклад, Header).
- `project/` — компоненти, пов'язані з управлінням проектами.
- `tasks/` — компоненти для роботи із задачами.
- `ModalWrapper.vue` — універсальний компонент модального вікна.
- `Table.vue` — універсальний компонент таблиці, який використовується для відображення як проектів, так і задач.

Такий поділ дозволяє:
- уникнути дублювання коду;
- спростити підтримку та масштабування проєкту;
- забезпечити повторне використання компонентів.

### `stores/`
Логіка роботи з даними винесена в окремі Pinia stores:
- `projects.ts` — управління проектами;
- `tasks.ts` — управління задачами.

Це дозволяє відокремити бізнес-логіку від UI-компонентів.

### `composables/`
Повторно використовувана логіка винесена в composables:
- `useTableFilters.ts` — фільтрація таблиць;
- `useTableReorder.ts` — зміна порядку елементів.

### `pages/`
Сторінки відповідають за композицію компонентів:
- `projects.vue` — список проектів;
- `projectDetail.vue` — детальна інформація про проект та його задачі.

## Основні принципи

- кожен компонент відповідає лише за одну функціональність.
- UI, бізнес-логіка та робота з даними розділені між компонентами, composables та stores.
- загальні компоненти та composables перевикористовуються в різних частинах застосунку.
- структура дозволяє легко додавати нові модулі та функціональність.
